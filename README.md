## 🎯 项目简介

一套跑在**飞书（Lark）**上的多智能体自动化系统。底层对接 DeepSeek/Qwen 等异构大模型集群，上层是可自编排的工具链。

**核心设计哲学**：每一层出问题不往上抛，自己兜住。

---

## 🏗 四层闭环架构

```
感知层 → 推理层 → 执行层 → 沉淀层
  ↑____________________________↓
        闭环反馈 · 知识反哺决策
```

| 层级 | 职责 | 核心技术 |
|------|------|----------|
| **感知层** | 多模态输入网关 | 格式自动判别、自适应提取、环境依赖自探测 |
| **推理层** | 长链任务规划 + 动态路由 | 三级容错链（主力→备用→兜底）、毫秒级切换 |
| **执行层** | 可复用技能库 + 跨会话记忆 | 工具链编排、Skill 固化、持久记忆复用 |
| **沉淀层** | 群聊知识蒸馏引擎 | 语义聚类、自动去噪摘要、月度全量回溯 |

## 🔧 工程亮点

### 文档结构保真引擎
逆向 OOXML `numbering.xml` 规范，独立编号注册机制，确保跨格式源到标准 Word 文档的结构 **100% 保真**。

### 三级模型容错链
```
主力推理节点 (DeepSeek-V4) → 备用集群 (Qwen3-8B) → 兜底节点 (V4-Free)
```
毫秒级自动切换，已持续运行 **30 天+**，经历多次主节点故障，零人工介入恢复。

### 敏感数据本地清洗
涉及金额、个人信息的原始数据在本地完成脱敏后才进入云端推理链路，架构层面预留私有化与混合部署可能。

## 🛠 技术栈

- **前端演示**：HTML5 + CSS3 + JavaScript（原生，无框架依赖）
- **动画**：CSS Animation / Keyframes / 3D Transform / cloneNode 重置
- **推理引擎**：DeepSeek-V4-Flash / Qwen3-8B / 三级容错链
- **OCR**：rapidocr-onnxruntime（纯 Python，零系统依赖）
- **文档转换**：OOXML 逆向 + python-docx
- **消息接入**：飞书 WebSocket 长连接
- **知识存储**：本地文件系统 + 持久记忆模块
